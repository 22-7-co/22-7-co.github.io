> **前言** 在 玩Linux时 （尤其是折腾 Arch、Niri、Hyprland 等 Wayland 桌面时），配置文件（Dotfiles）就是我们的身家性命。随着系统越来越定制化，单纯的复制粘贴已经无法满足我们在多套配置（比如自写极简版与开源套件版）之间灵活切换的需求。
> 
> 这次借着折腾新环境的机会，我全面引入了 `GNU Stow` 来接管我的 dotfiles。中间踩了不少坑，特此记录，希望能帮到同样在折腾桌面的你。

## 一、 为什么是 Stow？

在没有 Stow 之前，我一直不知道怎么管理配置文件，想试试别人的好看桌面，但是又怕自己的配置被覆盖，或者丢失。即使有btrfs，但折腾起来很麻烦。然而，利用Stow完美解决了这一点。

**GNU Stow 的核心哲学是“偷天换日（软链接）”。**

我们只需要在用户目录建一个“配置目录”（比如 `~/.dotfiles`），把所有的真实配置按照系统的目录层级塞进去，然后用 Stow 建立软链接，映射回 `~/.config` 里。

- **隔离性**：系统依然去 `~/.config/niri` 找配置，但它不知道这只是一个指向 `dotfiles` 的传送门。
- **版本控制**：Git 只需要在 `~/dotfiles` 这个纯净的文件夹里工作。
- **一键换肤**：随时撤销软链接，挂载另一套配置。

## 二、 基础实战：从零开始映射配置

假设我们要管理 Niri 的配置。

```
yay -S stow # 安装stow，不太系统命令不一定相同
```
### 1. 建立配置目录与层级结构

Stow 的规则是：**你的收纳目录结构，必须和目标目录结构严格一致。**
```
.dotfiles/
├── base/  # 自己命名，相当于不同配置
│   └── .config/  # 投射从~开始的本地目录
├── dms/
│   └── .config/
├── .git/
│   ├── gk
│   ├── hooks
│   ├── info
│   ├── logs
│   ├── objects
│   └── refs
├── inir/
│   └── .config/
└── niri/
    └── .config/
```
### 2. 转移真实文件

把现在正在用的配置“连锅端”移进来：
通过 `mv` 移动到 `~/.dotfiles`下，放心，就算桌面配置丢了，系统也不会崩溃。
假如你现在的配置是 `niri`, 那就放在 `~/.dotfiles/niri`下
### 3. 执行 Stow 

在 `~/.dotfiles`目录执行 `stow niri`，这个命令会自动根据层级关系将配置挂载到对应目录：
此时执行 `ls -l ~/.config/niri`，如果看到 `->` 箭头指向你的 dotfiles 目录，恭喜，第一步成功了。

### 4. 切换桌面

通过 `stow -D niri` 命令可以取消挂载，这时候，你可以再挂载别的桌面配置，比如`stow end4`, 当前stow是很灵活的，会了这些操作，你可以去直接 `copy` 别人的`.dotfiles`了, 而不用担心配置啦。

### 5. 通过 git 管理

通过`git` 来管理你的配置，防止丢失和损害，也方便分享给朋友。
