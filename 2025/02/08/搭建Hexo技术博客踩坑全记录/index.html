注：今天与Deepseek聊天的时候，经他介绍开始搭建了dexo的个人博客，其中踩了不少坑，打算写这篇文章来记录一下。

1. 配置GitHub Pages时仓库名不匹配：

  • GitHub仓库必须是 [你的用户名].github.io（例如 UnityDev2024.github.io）

2. 配置_config.yml时仓库地址填错

  • 正确的 repo 格式 repo: https://github.com/你的用户名/你的用户名.github.io.git
  • 而我的 repo 格式 repo：https://github.com/你的用户名.github.io.git`
  • 这就导致了一直找不到地址，部署不上去

3. 还有一些YAML语法规则相关的注意点

  • 冒号 : 后必须加空格（如 type: git ✅，type:git ❌）
  • 缩进使用空格（不要用Tab键）
  • 层级通过缩进控制（如 deploy 下的 type/repo/branch 需缩进两空格）

4. 执行 hexo 指令时必须进入 Hexo博客的根目录

    • 必须进入 Hexo 博客的根目录（即包含 _config.yml 的文件夹）才能执行命令

5. 创建批处理脚本和hexo重名了

  • 为了方便进入hexo根目录，我写了批处理脚本，但是命名和dexo重复了
  • 这会导致我的 hexo 指令执行失效
  • 然后我将 .bat 改成了 run hexo.bat

6. 运行结束时要正确关闭，要不然会导致端口被占

  • 运行结束后按 Ctrl + C 终止 hexo
  • 如果忘了关，需在任务管理器搜node，并把所有运行的都关闭

其他

  • 几个比较重要的指令
  • hexo clean && hexo generate && hexo deploy
  • 分别是清理缓存，生成静态文件，部署到GitHub
  • hexo server
  • 运行 Hexo服务
